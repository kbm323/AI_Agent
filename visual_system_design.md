### 1.2 FFmpeg 구현 템플릿 파이프라인

```bash
# === 기본 템플릿 구조 (파이프라인) ===

# Step 1: 원본 클립을 9:16으로 크롭/스케일
ffmpeg -i input_clip.mp4 \
  -vf "crop=ih*9/16:ih:(iw-ih*9/16)/2:0,scale=1080:1920" \
  -c:v libx264 -preset fast clip_916.mp4

# Step 2: 로고 오버레이 (좌측 상단, 페이드인)
ffmpeg -i clip_916.mp4 -i logo_lt_twins.png \
  -filter_complex "[0][1]overlay=20:15:enable='between(t,0,3)'" \
  -c:v libx264 logo_overlay.mp4

# Step 3: 하단 그라데이션 자막존 생성
ffmpeg -i logo_overlay.mp4 \
  -vf "drawbox=x=0:y=1400:w=1080:h=320:color=black@0.7:t=fill,
       drawtext=fontfile=Pretendard-Bold.ttf:text='자막 예시':fontcolor=white:fontsize=48:
         x=(w-text_w)/2:y=1480:enable='between(t,0,10)':alpha='if(lt(t,0.5),0,if(lt(t,1.0),(t-0.5)/0.5,1))'" \
  -c:v libx264 sub_overlay.mp4
```

### 1.3 전환효과 라이브러리

| 효과명 | FFmpeg 구현 | 지속시간 | 용도 |
|--------|------------|---------|------|
| **슬램 컷** | `select` + `trim` concatenation | 0.15s | 경기 하이라이트 빠른 전환 |
| **플래시 화이트** | `geq`로 화이트 프레임 2~3f 삽입 | 0.1s | 홈런/득점 순간 |
| **블러 인/아웃** | `boxblur` + `overlay` | 0.3s | 구간 전환 |
| **줌 바운스** | `zoompan` with overshoot | 0.5s | 선수 등장, 주요 순간 |
| **슬라이드 업** | `overlay` with animated y | 0.4s | 자막 교체 |
| **글리치** | `hue` shift + `eq` + `noise` | 0.2s | 팬덤 밈, 랩/힙합 비트 |
| **페이드 투 블랙** | `fade` filter | 0.5s | 구간 끝, 회상 |

**FFmpeg 페이드 인/아웃 명령어 예시**:
```bash
# 페이드 인 (처음 0.5초)
ffmpeg -i input.mp4 -vf "fade=t=in:st=0:d=0.5" output.mp4

# 페이드 아웃 (마지막 0.5초)
ffmpeg -i input.mp4 -vf "fade=t=out:st=9.5:d=0.5" output.mp4

# 크로스페이드 (두 클립 연결)
ffmpeg -i clip1.mp4 -i clip2.mp4 \
  -filter_complex "[0][1]xfade=transition=fade:duration=0.3:offset=4.7" output.mp4
```

### 2.2 ⚠️ Python/스크립트 보조 필요

| 요소 | 한계 | 해결책 |
|------|------|--------|
| **복잡한 모션 그래픽** | `drawtext` 표현식으로 복잡한 애니메이션 구현 난해 | Python으로 SRT+ASS 파일 동적 생성 후 `subtitles` 필터 사용 |
| **다이나믹 레이아웃** | 필터 체인 길어지면 관리 어려움 | Python으로 필터 그래프 동적 생성 |

| **점수판 오버레이** | 실시간 데이터 연동 | Python 템플릿 → PNG → `overlay` |
| **3D 텍스트/카드플립** | FFmpeg으로 불가능 | Blender CLI 렌더링 or 제외 |
| **이모지/특수문자** | 폰트 의존성 높음 | Apple Color Emoji 폰트 번들링 |

| 계층 | 폰트 | 두께 | 용도 |

| **H2 (서브 타이틀)** | Pretendard Bold | 700 | 선수 이름, 구간 타이틀 |

### 3.4 FFmpeg drawtext 애니메이션 레시피

```bash
# === 1. 팝인 자막 (0→1.2x→1x 바운스) ===
# 0.3초 동안 팝인
drawtext="text='GOOOAL!':fontfile=Pretendard-Black.ttf:fontsize=64:
  fontcolor=white:x=(w-text_w)/2:y=1480:
  fontsize='if(lt(t,0.3), 
    64*0.1+64*0.9*sin(PI/2*t/0.3),
    64)':
  alpha='if(lt(t,0.3),(t)/0.3,1)'"

# === 2. 타이핑 효과 (한 글자씩 등장) ===
# Python에서 문자별 enable 타이밍 계산 후 여러 drawtext 연결
# or ASS 파일로 구현

# === 3. 슬라이드 업 자막 (아래→위로 등장) ===
drawtext="text='한 방이면 충분하다':...:
  y='if(lt(t,0.5), h+100 - (h+100-h*0.78)*(t/0.5), h*0.78)'"

# === 4. 워블/바이브레이션 (밈 효과) ===
drawtext="text='ㅋㅋㅋㅋ':...:
  x='(w-text_w)/2 + 5*sin(t*30)':
  y='h*0.78 + 3*cos(t*25)'"

# === 5. 글로우 텍스트 (텍스트 복제 + 블러) ===
# 2-pass: 블러 텍스트 먼저 → 선명한 텍스트 오버레이
-vf "drawtext=...:fontcolor=#C30452@0.5:shadowx=0:shadowy=0:shadowcolor=#C30452@0.3:borderw=6:bordercolor=#C30452@0.2,
     drawtext=...:fontcolor=white"
```

### 3.5 자동화 일관성 유지 방법

- **템플릿 JSON**: 모든 타이포그래피 사양을 `typo_config.json`에 정의
- **사전 검증**: Python 스크립트로 모든 스타일이 사양을 준수하는지 체크
- **스타일 프리셋**: `drawtext` 파라미터를 모듈화하여 코드 중복 제거
- **주간 스타일 감사**: 자동 생성된 쇼츠에서 랜덤 샘플링 → 규칙 준수 확인

### 4.2 FFmpeg 썸네일 생성 파이프라인

```bash
#!/bin/bash
# auto_thumbnail.sh — LG트윈스 쇼츠 썸네일 자동 생성

VIDEO="$1"
OUTPUT="$2"
TIMESTAMP="${3:-00:00:01.5}"  # 기본 1.5초 지점

# 1. 하이라이트 프레임 추출 (9:16 크롭)
ffmpeg -ss "$TIMESTAMP" -i "$VIDEO" \
  -vf "crop=ih*9/16:ih:(iw-ih*9/16)/2:0,scale=704:1280" \
  -vframes 1 -q:v 2 highlight_frame.png

# 2. 하이라이트 프레임을 배경으로 배치 (좌측 55%)
ffmpeg -i highlight_frame.png \
  -i thumbnail_bg_template.png \
  -filter_complex "[1][0]overlay=0:(H-h)/2" \
  thumbnail_base.png

# 3. 텍스트 오버레이 추가
ffmpeg -i thumbnail_base.png -i logo_lt_white.png \
  -vf "drawtext=fontfile=Pretendard-Black.ttf:text='오늘의 한방':
       fontcolor=white:fontsize=72:x=750:y=100,
       drawtext=fontfile=Pretendard-Bold.ttf:text='${PLAYER_NAME}':
       fontcolor=#C30452:fontsize=56:x=750:y=200,
       [in]overlay=900:20" \
  -vframes 1 "$OUTPUT"

# 4. YouTube Data API로 업로드 (Python 스크립트 활용)
# python upload_thumbnail.py --video-id $VIDEO_ID --file $OUTPUT
```

### 5.1 경기 클립 템플릿 (50%)

### 6.2 브랜드 요소 자동 삽입 체크리스트 (Python Validation)

```python
def validate_brand_compliance(video_path):
    """자동 생성된 쇼츠의 브랜드 컴플라이언스 검증"""
    checks = []

    # 1. 로고 존재 여부 (첫 1초)
    checks.append(check_logo_presence(video_path, timeframe=(0, 1.0)))

    # 2. 트윈스 레드 픽셀 비율 (최소 2%)
    checks.append(check_color_ratio(video_path, target_hex="#C30452", min_ratio=0.02))

    # 3. 엔딩카드 지속시간 (최소 3초)
    checks.append(check_ending_card_duration(video_path, min_sec=3.0))

    # 4. 자막 가독성 (대비율 4.5:1 이상)
    checks.append(check_subtitle_contrast(video_path, min_ratio=4.5))

    # 5. 금지 컬러 검출 (상대팀 컬러 오용)
    checks.append(check_forbidden_colors(video_path, forbidden=["#131230", "#000080"]))

    return all(checks), checks
```
